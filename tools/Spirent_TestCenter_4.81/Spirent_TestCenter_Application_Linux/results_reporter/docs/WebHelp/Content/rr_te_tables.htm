Template Editor tables are defined with a subset of standard HTML elements plus a new attribute in the tr element, iterate. Table definitions also use a form of variable substitution to specify where data from a table, a reported statistic or a computed statistic is to be displayed.

Variable Substitution

In a table definition, a variable name is indicated with a leading equal sign followed by the variable name delimited by dollar signs. For example, if the text =$DroppedFrameCount$ appears in a table cell definition, the value of the table column called DroppedFrameCount is displayed.

Cell Definition What is displayed in the table
<td>DroppedFrameCount</td> DroppedFrameCount
<td>=$DroppedFrameCount$</td> The value of the field DroppedFrameCount

If the variable doesn't exist, the variable name is displayed instead of a value. The Template Editor supports use of variable substitution in table cells, the bgcolor attribute, and the iterate attribute.

The iterate Attribute of the tr Element

The Template Editor introduces a new attribute. The iterate attribute is used in a table row definition to specify that multiple rows are to be generated, one for each item in a list. For example, the code

<tr iterate="=$PortName$">
<td> =$PortName$ </td>
<td> =$TxPkts$ </td>
<td> =$RxPkts$ </td>
</tr>

will generate one table row for each item in the PortName list. Each row will display a port name and the corresponding number of transmitted and received packets for the port. Variables in other columns (such as =$TxPkts$ and =$RxPkts$) should also be list items.

The iterate attribute can be used in combination with the bgcolor attribute and variable substitution to dynamically control the background color of a cell.

<tr>
<td bgcolor="ffffff">Traffic Group</td>
<td bgcolor="ffffff">Traffic Class</td>
<td bgcolor="ffffff"> Pass/Fail </td>
</tr>

<tr iterate="=$TrafficClass$">
<td bgcolor="ffffff"> =$TrafficGroup$ </td>
<td bgcolor="ffffff"> =$TrafficClass$ </td>
<td bgcolor="=$StatusColor$"> =$PassFail$ </td>
</tr>

This table will display the traffic group and test result (pass/fail) for each traffic class. The first two columns have a white background. The background color of the third column is determined by the computed statistic StatusColor, which is a list with a green color code (#0000ff) if the value of PassFail is "Pass" and a red color code (#ff0000) if the value of PassFail is "Fail". A sample rendering of the table could look like this:

Traffic Group Traffic Class Pass/Fail
TCP Class 1 Pass
UDP Class 2 Fail
None Class 3 Pass
