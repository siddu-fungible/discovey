Results Reporter Template Editor Overview

Spirent TestCenter Results Reporter features templates that incorporate tables and graphs to present test results. Template-based reports can be viewed on the screen and exported to PDF and HTML files. The Template Editor is a tool used to modify existing templates and create new templates.

Starting the Template Editor

The Template Edit can be run from a command line or launched inside Results Reporter.

To start the Template Editor from within Results Reporter

You can launch the Template Editor from within Results Reporter through the menu, toolbar or a keyboard shortcut, to edit an existing template or create a new one.

  • On the Results Reporter Tools menu, select Show Template Editor.
  • On the Results Reporter toolbar, click the Template Editor button.
  • Use the keyboard shortcut CTRL+E.

The Template Editor opens in the right window of Results Reporter.

  • To edit an existing template, open a database that uses the template, highlight the database node, select the template from the dropdown, then launch the Template Editor.
  • To create a new template, launch the Template Editor without selecting a template.

To start the Template Editor from the command line

You can launch the Template editor in standalone mode from the command line.

  • Command:
    • Windows: STCTemplateEditor.bat
    • Unix/Linux: STCTemplateEditor.sh
  • Path:
    • Windows: Spirent Communications\Spirent TestCenter n.nn\Spirent TestCenter Application\Results Reporter
    • Unix/Linux: <installed path>/results_reporter

The Template Editor opens with a new query and a blank page.

To close a template

Highlight the template in the Results Reporter tree and press DELETE.

The Template Editor Window

The Template Editor window is divided into horizontal zones.

Zone Description
Menu

Available when the Template Editor is started from the command line

Open database, open and save a template, generate PDF and HTML reports, and re-query a database

Toolbar Open databases, open and save templates, generate PDF and HTML reports, and re-query a database
Tabs

Configure the page layout, tables, graphs, stats, and view the database

  • When customizing an existing template, the typical working order is to start with the layout tab and work to the right as needed.
  • When creating a new template, the typical working order is to start with the DB Explorer tab and work to the left.
Tab Description
Layout Configure page layout by creating and editing text, images, graphs and tables
Tables Create and modify tables based on reported and computed statistics
Graphs Create and modify graphs based on reported and computed statistics
Reported Stats Select statistics from the results database
Computed Stats Compute statistics from the results database using Tcl code
DB Explorer

View database tables

The Raw tab shows the contents of the table

The Joined tab is a legacy feature that will be deprecated

Productivity tip: When you create a query in Results Reporter, the list of database tables is sorted by table name, making it easier to find a specific table. In addition, a Query field allows you to try out different SELECT statements and immediately see the results. Using Results Reporter queries allows you to prototype your queries quickly before creating them in the Reported Stats tab.
Workspace Configure template elements. The appearance of the workspace varies based on the tab selected.
