Overview

The OSPFv2 Scale Benchmark methodology measures the maximum number of simultaneous OSPF adjacencies the device under test can sustain.

Topology

The topology for this methodology, is shown in the diagram below, consists of

  • A device under test (DUT), which is an OSPFv2 router

  • Multiple OSPFv2 routers, simulated by Spirent TestCenter port(s) on both the “east” side and the “west” side of the DUT



Test Steps

The steps in this methodology are as follows:

  1. Conduct a binary search for the maximum number of OSPF adjacencies with the DUT, starting at the configured minimum number of adjacencies (simulated OSPFv2 routers).

  2. Attempt to establish the required number of adjacencies for the trial.

  3. For each trial, send traffic at the configured rate.

  4. If the OSPF adjacencies are stable throughout the trial, and traffic is received without loss, then the trial passes; otherwise it fails.

  5. Stop all simulated OSPFv2 routers.

  6. Adjust the number of simulated OSPFv2 routers either up or down, depending on the outcome of the trial.

  7. Repeat steps 2 through 6 until the maximum stable simulated router count is found.

  8. If the maximum stable adjacency (session) count exceeds, or is the same as, the expected sessions per port, then the test passes; otherwise it fails.
