Overview

The RFC 2889 Address Learning methodology implements section 5.8 of the RFC 2889 IETF standard for LAN switch benchmarking. The purpose of this methodology is to determine the rate of address learning of a LAN switching device.

In this methodology, Spirent TestCenter ports are used to send and receive all frames. The basic algorithm of the test is to do a binary search across a range of learning rates until the maximum learning rate is found. For each iteration, learning frames are initially sent from the Learning port(s), through the SUT, and on to the Test port(s); then test frames are sent back from the Test port(s), back through the SUT and to the Learning port(s). If the SUT floods frames to the Monitor ports, the attempt was not successful. The attempted learning rate is adjusted up or down depending on whether the trial was successful or not, i.e. the SUT was able to “learn” the addresses such that it could successfully forward test frames. The test is repeated for each of the specified frame sizes.

The methodology can work without VLANs, with single VLANs or stacked (QinQ) VLANs.

  • Test Ports, which receive learning frames from the SUT, and transmit back test frames through the SUT to the Learning ports

  • Monitor Port(s), which potentially receive overflow frames from the SUT

Test Steps

The steps in this methodology are as follows:

  1. For each frame size, do all of steps below.

  2. For the configured number of trials conduct a binary search for the maximum address learning rate, as follows

    1. Starting at the Initial Learning Rate, send learning frames from the Learning ports to the SUT.

    2. After all learning frames are sent, send test frames from the Test ports to the SUT.

    3. If all test frames are successfully forwarded to the Learning ports, without broadcast frames overflowing to the Monitor port, the iteration is successful.

    4. If successful, the learning rate should be increased to the mid-point between the initial learning rate and the maximum learning rate; if unsuccessful, the learning rate should be decreased to the mid-point between the initial learning rate and the minimum learning rate.

    5. Wait for the specified aging time to expire so that the SUT has “forgotten” any addresses it learned in the trial.

    6. Repeat steps a through d above until the measured learning rate is determined within the resolution specified in the configuration.

  1. The test passes if,

    • The maximum measured learning rate is greater than the Address Learning Rate Test Criteria (in frames per second).
