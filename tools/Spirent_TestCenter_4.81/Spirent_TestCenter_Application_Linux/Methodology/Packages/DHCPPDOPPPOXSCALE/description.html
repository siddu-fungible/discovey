Overview

DHCP Prefix Delegation is an IPv6 technology that assigns network address prefixes to client networks. The DHCP-PD over PPPoE Session Scale methodology determines the maximum number of DHCP-PD over PPPoE sessions that can be established.

In this methodology, Spirent TestCenter emulates multiple DHCP-PD clients which request prefixes from the DUT. Each setup attempt consists of first establishing a PPPoE session, followed by a prefix request. The methodology starts by establishing sessions at a minimum scale; if successful, the scale is increased until either the maximum number of sessions is established, or the SUT will not accept any more sessions. For highest achieved session scale, if selected, traffic is sent from the traffic endpoint ports to the DHCP Clients, after the DHCP sessions are established.

This methodology works with IPv4, IPv6, or both. The methodology can also work with single or stacked (QinQ) VLANs.

Topology

  • A DHCP Client Port Group, where Spirent TestCenter ports emulating multiple PPPoE Clients

  • The SUT, which is an access device, potentially including a DHCP/PPP server

  • A DHCP Server/Traffic Endpoint port group

Test Steps

The steps in this methodology are as follows:

  1. Perform a binary search to find out the maximum scale at which sessions can be established, with each step being as follows:

    • Establish a PPP session

    • Request a prefix to be delegated

  2. Send traffic from the traffic endpoint ports, through the SUT, to the DHCP client ports.

  3. The test passes if,

    • the achieved session scale is greater than or equal to the configured threshold, within the configured tolerance, and

    • traffic is received at greater than or equal to the configured threshold rate on all receive test ports
