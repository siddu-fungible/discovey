Overview

The Ping Methodology verifies connectivity through a DUT using ICMP messages.

With each trial, ping request (ICMP echo request) messages are sent from STC test ports in the west port group, through the DUT, to STC test ports in the east port group. After receiving ping request messages, the east port group test ports reply with ping response (ICMP echo response) messages, which are transmitted back through the DUT to the west port group test ports.

Ping test packets for this test may be defined only as IPv4. Inner and outer VLAN IDs may also be configured for each test port.

Test Steps

  1. For each configured port, send a ping request message with the specified ping source address to the specified ping destination address.

  2. For each ping request message received, reply with a ping response message

  3. This test passes if ping responses are received for all ping requests.
