Overview

The BGP Route Reflector IPv6 Route Scale methodology measures the time it takes for a DUT route reflector to re-advertise a collection of IPv6 routes. Routes used by this methodology are imported from a text file.

Routes may originate from either client or non-client peers, and are expected to be reflected as follows:

  • A route from a Non-Client peer: reflect to all Clients only

  • A route from a Client peer: reflect to all Clients and Non-Clients

Topology

The topology for this methodology, is shown in the diagram below, consists of

  • A route reflector, which is the device under test

  • Multiple PE client routers, simulated by Spirent TestCenter port(s)

  • Multiple PE non-client routers, simulated by Spirent TestCenter ports



Test Steps

The steps in this methodology are as follows:

  1. All i-BGP peers are brought up simultaneously

  2. Start to advertise routes from all client and non-client peers on STC test ports

  3. The DUT/Route Reflector should reflect all advertised routes.

  4. Receive the re-advertised routes is indeed received on each simulated client and non-client emulating STC test port.

  5. Verify that the received routes are

    • At least as many as expected

    • Not greater than the expected tolerance

    • Received within the specified time limit

  6. The test passes if all of the above conditions are met; otherwise the test fails.

  7. Stop all BGP sessions
