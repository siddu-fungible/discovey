Overview

The RFC 2889 Broadcasting Frame Forwarding methodology implements the throughput test portion of section 5.10 of the RFC 2889 IETF standard for LAN switch benchmarking. The purpose of this methodology is to determine the throughput of the SUT while forwarding broadcast frames.

In this methodology, Spirent TestCenter ports are used to send and receive all frames. The basic algorithm of the test is to generate test traffic with a broadcast address, at the intended load (ILoad) rate, from a single Source port to the SUT, which forwards the test traffic on to the Receive port(s) at the offered load (OLoad) rate. The latency through the SUT is measured. The exercise is repeated for the specified number of trials, for each frame size. The forwarding rate for all frame sizes is expected to be greater than the configured forwarding rate test criteria.

The methodology can work without VLANs, with single VLANs or stacked (QinQ) VLANs.

Test Steps

The steps in this methodology are as follows:

  1. For each of the specified frame sizes, do all of the steps below.

  2. For configured number of trials, do all of the steps below

  3. Starting at the lower limit, send traffic from the Source Port to the SUT for the configured duration

  4. On all Receive port(s), measure the throughput.

  5. If the measured throughput on all receive ports is greater than the configured threshold, increase the traffic rate to the mid-point between previous successful iteration and the upper limit; if the measured throughput is less than the configured threshold, decrease the traffic rate to the mid-point between the previous successful iteration and the lower limit.

  6. Repeat from step 3 onwards until the successful rate is determined within the configured resolution.

  7. The test passes if the highest achieved throughput is greater than or equal to the forwarding rate test criteria, within the specified resolution.
