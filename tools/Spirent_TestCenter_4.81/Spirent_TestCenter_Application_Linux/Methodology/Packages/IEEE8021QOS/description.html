Overview

This methodology validates QoS operation using the Class of Service (CoS) bits in the frame definition specified in IEEE 802.1p and codified in the PRI field of VLAN tags as specified in IEEE 802.1Q. The test creates situations in which there are an increasing number of VLANs and a mix of PRI traffic. The test validates that the SUT properly prioritizes the appropriate flows and delivers on its QoS guarantees when supporting a large number of VLANs. The test has multiple iterations, each with a fixed frame size for all traffic.

The test presents the SUT with traffic at every PRI level shown in Table 1. The traffic will be presented to multiple ingress ports on the SUT. The traffic loads will be controlled such that one or more egress ports on the SUT experience congestion. When congestion occurs, the SUT will drop frames. The test will validate that the higher-priority traffic will continue to be delivered per QoS requirements and that lower-priority traffic will experience degradation and drops.

Topology

The topology for this methodology is shown in the diagram below, consists of the following port groups:

  • Tx Load Port Group (TxLoad): Emulates source endpoints for PRI 0-6 traffic (7 streams per port)

  • Tx Test Port Group (TxTest): Emulates source endpoints for PRI 7 traffic (1 stream per port)

  • Rx Port Group (Rx): Emulates destination endpoints for PRI 0-7 traffic (8 streams per port)

Traffic will be unidirectional. Each traffic stream will originate on devices at either TXLoad or TXTest. All traffic streams will terminate on devices at RX.

TXLoad is configured such that it delivers a constant 7/8 (87.5%) saturation load to RX. As TXTest is increased in steps of 1/8 (12.5%) saturation load to RX we expect to see the SUT drop traffic from the lowest priority levels. For example, when TXLoad is providing 87.5% RX saturation and TXTest is providing 25% RX saturation we expect PRI 0 (and only PRI 0) traffic to be dropped by the SUT.

Test Steps

  1. Load the configuration for the test. This includes the SUT and the testing tool.

  2. If learning is enabled, perform L2 learning and populate configuration as required.

  3. Start sanity run. This is Tx Load traffic only. TxTest traffic is set to 0%. Run for one iteration length of time to validate setup.

  4. Validate that the performance of the SUT is according to the values listed in Table 1.

  5. For each frame size, perform all of the steps below.

  6. Loop through iterations of successively increasing TxTest loads, stepping through TxLoad values that present the following PRI 7 loads to the Rx ports:
