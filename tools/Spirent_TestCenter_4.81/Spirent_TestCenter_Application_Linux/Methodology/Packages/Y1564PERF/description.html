Overview

The Y.1564 Service Performance methodology implements the service performance test specified in the ITU-T Y.1564 standard, “Ethernet service activation test methodology”, which specifies procedures for validating a SUT’s ability to transport services based on set of acceptance criteria in a service level agreement (SLA).

The Y.1564 service performance methodology tests all services simultaneously at the committed information rate (CIR)

Acceptance criteria for each service are specified by three parameters:

  • FLR (frame loss ratio)

  • FTD (frame transfer delay)

  • FDV (frame delay variation)

In this methodology, Spirent TestCenter ports are used to generate service traffic, as well as measure received traffic. Spirent TestCenter also measures the FLR, FTD and FDV of the received service traffic, and compares the measured values against the acceptance criteria for each service

This methodology works with IPv4, IPv6, or both. The methodology can also work with single or stacked (QinQ) VLANs.

Test Steps

The steps in this methodology are as follows:

  1. Define a set of stream blocks, one per test service, with the attributes as configured in the methodology.

  2. Start the Service Performance test, if selected by the user

    1. Transmit traffic for all services, simultaneously.

    2. On the each of the specified receive ports for each service, measure the FLR, FTD, and FDV.

    3. If the maximum measured FLR, FTD or FDV exceeds the acceptance criteria for any service, fail the service performance test.

  1. The test passes if,

    • All service configuration sub-tests pass

    • The service performance test passes
